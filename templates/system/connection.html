{# Troubleshooting #}
Troubleshooting
Staff can't reach the server from their devices
  1. Confirm all devices are on the same Wi-Fi or LAN.
  2. Share the Fallback URL above (http://{{ lan_ip|default:"192.168.x.x" }}:8000).
  3. If that fails, check Windows Firewall — allow TCP port 8000 inbound.
  4. Confirm Waitress is running with --host=0.0.0.0 (not 127.0.0.1).
Camera not working on client devices
Browsers require HTTPS (or localhost) for camera access. Client devices must use {{ server_domain_url }} via Caddy. The fallback HTTP URL will never allow camera access — this is a browser restriction.
Login fails with "403 Forbidden" or "CSRF cookie not set"
This happens when SECURE_COOKIES=True (or default with DEBUG=False) and the user accesses via plain HTTP. Fix: add SECURE_COOKIES=False to .env, or direct users to the HTTPS domain.
Domain (fans-barangay.local) not resolving on client devices
Add the following line to C:\Windows\System32\drivers\etc\hosts on each client device (requires admin rights):
{{ lan_ip|default:"<server-ip>" }}  fans-barangay.local
"Bad Request (400)" when accessing via HTTPS
CSRF_TRUSTED_ORIGINS must include https://fans-barangay.local in .env. Also check SECURE_PROXY_SSL_HEADER and USE_X_FORWARDED_HOST are set correctly.
